**Среда исполнения:** n8n Cloud workflow «КотЭ — AI Агент с памятью»
**Webhook URL:** `https://ibetekhtin.app.n8n.cloud/webhook/kote/webhook`

### Pipeline КотЭ (n8n)

```
📱 Telegram Trigger
    ↓
📝 Извлечь        — фильтр ботов, извлечь tg_chat_id / message_text / name
    ↓
👤 Upsert клиент  — RPC bot_upsert_client → создать/обновить клиента в БД
    ↓
🧠 Загрузить контекст — RPC get_kote_context → история, туры, знания, память
    ↓
✍️ Собрать промпт — сборка системного промпта + is_new_client
    ↓                        ↓
🤖 Gemini API         🆕 Новый клиент? ──(true)──▶ 📲 Уведомить менеджера
    ↓
💬 Достать ответ  — извлечь текст + обработка ошибок API
    ↓                        ↓
📤 Отправить          💾 Сохранить диалог (→ conversations)
(ответ клиенту)               ↓
                      🔍 Интент — определить намерение из сообщения
                            ↓                    ↓
                    ❓ Есть обновления?    🔔 Бронь? ──(true)──▶ 📲 Менеджер
                      ↓         ↓
               📊 Память   🎯 Стадия
```

### Воронка клиента
`new → interest → thinking → booking → done`

### RPC функции (SECURITY DEFINER, доступны через anon key)

| Функция | Параметры | Назначение |
|---------|-----------|-----------|
| `bot_upsert_client` | tg_chat_id, name, source | Создать/обновить клиента |
| `get_kote_context` | tg_chat_id, query, secret | Полный AI-контекст (секрет: KOTE_SECRET) |
| `search_knowledge` | query | Поиск по базе знаний |
| `upsert_client_memory` | client_id, interests, budget_level, ... | Обновление памяти |
| `update_client_stage` | tg_chat_id, stage | Движение по воронке |
| `app_upsert_lead` | ... | Единая точка записи лида (сайт + API) |
| `get_new_leads` | p_minutes=6 | Новые лиды за N минут (для n8n) |
| `get_tour_reminders` | p_days_ahead=1 | Брони на завтра (для n8n) |
| `get_review_requests` | p_days_ago=0 | Завершённые туры (для n8n) |

### Безопасность (RLS)
- `get_kote_context`, `upsert_client_memory`, `update_client_stage` — только через RPC (SECURITY DEFINER)
- `app_upsert_lead`, `search_knowledge`, `bot_upsert_client`, `get_new_leads` — доступны anon
- `conversations` — anon INSERT через политику `conversations_anon_insert_site`
- `client_memory` — только через RPC (прямой anon-доступ закрыт)
