## 🤖 Workflow 1: КотЭ — AI Агент с памятью (основной)

### Статус: ✅ Активен

### Архитектура (13 нод)

```
📱 Telegram trigger
    ↓
📝 Извлечь (Code: парсинг сообщения)
    ↓
👤 Upsert клиент (Supabase RPC: app_upsert_lead)
    ↓
🧠 Загрузить контекст (Supabase RPC: get_kote_context)
    ↓
✍️ Собрать промпт (Code: формирование контекста)
    ↓
🤖 Gemini (HTTP: прямой вызов Google API)
    ↓
💬 Достать ответ (Code: парсинг ответа)
    ↓
📤 Отправить (Telegram) + 💾 Сохранить диалог (Supabase)
    ↓
🔍 Интент (Code: определение намерений клиента)
    ↓
❓ Есть обновления? (IF)
    ↓
📊 Обновить память (Supabase RPC) + 🎯 Обновить стадию (Supabase RPC)
```

### Что работает отлично
- ✅ Память клиента (interests, budget, arrival date, group size)
- ✅ Интент-детекция (16+ паттернов: islands, active, nature, evening, family, vip)
- ✅ Определение стадии клиента (new → interest → thinking → booking → done → cold)
- ✅ Извлечение даты приезда из текста
- ✅ Определение размера группы и наличия детей
- ✅ Сохранение истории диалога
- ✅ Каскадное обновление памяти и стадии

### ⚠️ Проблема: НЕТ FALLBACK для AI

**Текущее:** Workflow вызывает Gemini **напрямую** через HTTP (Google API).
**Проблема:** Если Gemini упадёт — бот не ответит. Нет fallback на AITUNNEL/Groq/OpenRouter.

**Решение (рекомендация):**
Заменить прямой вызов Gemini на вызов FastAPI бэкенда:
```
POST https://localhost:8000/api/v1/ai/ask
{
  "prompt": "...",
  "system": "...",
  "max_tokens": 600,
  "temperature": 0.85
}
```
Это даст автоматический fallback через цепочку AITUNNEL → Groq → OpenRouter → Gemini.

**Альтернатива (проще, но без fallback):**
Оставить как есть, но заменить Gemini на AITUNNEL (тот же OpenAI-совместимый формат).

### Что можно улучшить
1. **Добавить fallback** — вызывать через FastAPI вместо прямого Gemini
2. **Обработка ошибок** — текущая обработка в «💬 Достать ответ» хорошая, но нет retry
3. **Таймаут** — нет таймаута на HTTP-запрос к Gemini

## 🔍 Найденные проблемы

### 1. Прямой вызов Gemini (без fallback)
- **Критичность:** Средняя
- **Описание:** n8n workflow вызывает Gemini напрямую, минуя fallback chain
- **Решение:** Переключить на FastAPI `/api/v1/ai/ask` или заменить на AITUNNEL

### 2. Placeholder ключей в workflow.json
- **Критичность:** Низкая (файл — документация)
- **Описание:** `SUPABASE_ANON_KEY_ЗДЕСЬ` — placeholder, не реальный ключ
- **Решение:** При импорте заменять вручную

### 3. daily-report.json невалидный
- **Критичность:** Низкая
- **Описание:** Невалидный `$supabase` expression
- **Решение:** Не импортировать, переписать при необходимости

### 4. Нет rate limiting на webhook
- **Критичность:** Низкая
- **Описание:** Webhook-эндпоинты n8n не защищены rate limiting
- **Решение:** Добавить rate limiting в nginx для n8n webhooks
